# 基于毫米波雷达点云的步态识别实现

## 摘要
步态识别是一种非接触、可远距离工作的生物特征识别技术，在智能安防、智慧养老和人机交互等场景具有重要应用价值。与传统视觉方案相比，毫米波雷达在弱光环境和隐私保护方面具有天然优势。本文围绕“基于毫米波雷达点云的步态识别实现”开展研究，基于现有点云样本数据构建了一套可复现的训练、评估与预测工程流程。方法上，本文采用纯雷达单模态技术路线，以点云序列为输入，首先进行统一规格化处理，再利用 PointNet 提取空间几何特征，使用 LSTM 建模时序动态规律，最后通过 Mean Pooling 聚合序列特征并结合 Triplet Loss 与 CrossEntropy Loss 联合优化。系统实现了 train/eval/predict/predict_batch 四种运行模式，并输出 Rank-1、mAP 等指标用于性能评估。实验结果表明，该方法能够完成30名受试者数据上的身份区分任务，具备较好的工程可用性与可扩展性。本文同时分析了当前方案在稳定性、类别混淆和参数敏感性方面的不足，并给出后续优化方向。研究结果可为毫米波雷达步态识别的工程落地提供参考。

**关键词**：毫米波雷达；点云；步态识别；深度学习；身份识别

## Abstract
Gait recognition is a non-contact biometric technology that supports long-distance identification and is widely useful in intelligent security, elder care, and human-computer interaction. Compared with vision-based methods, millimeter-wave radar has natural advantages in low-light robustness and privacy preservation. This thesis focuses on gait recognition using millimeter-wave radar point clouds and builds a reproducible engineering pipeline based on existing point-cloud sequence samples. The proposed radar-only framework takes point-cloud sequences as input, applies unified input normalization, extracts spatial geometric features with PointNet, models temporal dynamics with LSTM, and aggregates sequence-level representations by Mean Pooling. The model is optimized jointly with Triplet Loss and Cross-Entropy Loss. The system provides four executable modes (train/eval/predict/predict_batch) and reports Rank-1 and mAP for quantitative evaluation. Experimental results on 30 subjects show that the method can effectively perform identity discrimination and has good engineering practicality and extensibility. Finally, this work discusses current limitations, including stability fluctuation, class confusion, and parameter sensitivity, and proposes future optimization directions.

**Keywords**: millimeter-wave radar; point cloud; gait recognition; deep learning; person identification

## 1 绪论
### 1.1 研究背景与意义
步态识别通过人体行走模式实现身份判别，具有“无需配合、远距离、非接触”的优势。传统视觉步态识别在光照变化、遮挡和隐私保护方面存在明显局限。毫米波雷达可在弱光甚至无光环境下稳定工作，且采集信息主要表现为距离、速度与角度相关特征，不直接暴露人脸与外观细节，适用于隐私敏感场景。基于毫米波雷达点云开展步态识别研究，对提升复杂环境下身份识别的可靠性与可部署性具有现实意义。

### 1.2 国内外研究现状
现有研究主要集中在三类方向：
1. 基于微多普勒谱的步态识别，具有较强速度信息表达能力，但空间结构利用不足。  
2. 基于点云的时空建模方法，可同时利用几何与动态信息，已成为研究热点。  
3. 多模态融合方法（点云+速度+信号强度/图像等），在部分场景下能提升精度但系统复杂度更高。  

在点云建模方面，PointNet 解决了点云无序输入问题；后续研究通过局部结构建模、时序建模与注意力机制持续提升性能。结合本文任务目标与工程可实现性，本文采用“雷达单模态 + 时空建模”的技术路线。

### 1.3 本文研究内容与创新点
本文主要工作如下：
1. 构建毫米波点云步态识别工程流程，实现训练、评估、预测一体化。  
2. 设计 Radar-Only 网络框架：PointNet（空间）+ LSTM（时序）+ Mean Pooling（聚合）。  
3. 采用 Triplet Loss + CrossEntropy Loss 联合优化，提高特征判别能力与训练稳定性。  
4. 增加批量随机预测统计模式，实现终端直接输出样本级结果与整体准确率。  

### 1.4 论文结构安排
第1章介绍研究背景与问题定义；第2章介绍理论基础；第3章给出系统方案；第4章说明工程实现；第5章开展实验与分析；第6章总结全文并提出后续改进方向。

## 2 相关理论与关键技术
### 2.1 毫米波雷达点云表征
毫米波雷达点云可表示为离散点集合，每个点通常包含位置与运动相关信息。在本文工程中，单点特征采用4维表示（如 x、y、z、doppler），样本为时序张量。点云具有无序性、稀疏性和噪声敏感性，需要结合网络结构和规范化策略进行建模。

### 2.2 点云深度学习方法
PointNet 通过共享 MLP 与对称聚合函数（如 max pooling）对无序点集进行特征抽取，避免对点顺序的依赖。其优势是结构简洁、工程实现稳定，适合作为基线空间特征提取器。

### 2.3 时序建模方法
LSTM 能够对跨帧序列中的动态变化进行建模，适用于步态这种明显具有周期性与时序性的任务。通过将逐帧空间特征输入 LSTM，可获得时序上下文表示。

### 2.4 度量学习与分类联合优化
Triplet Loss 强调“同类近、异类远”，提升特征嵌入空间判别性；CrossEntropy Loss 提供明确身份监督，促进模型收敛。联合优化兼顾检索与分类能力，是当前工程中常用且有效的训练策略。

## 3 系统方案设计
### 3.1 总体技术路线
本文技术路线如下：  
数据输入（.npy）→ 输入规格化（20帧×80点×4维）→ PointNet 空间特征提取 → LSTM 时序建模 → Mean Pooling 序列聚合 → Triplet+CE 联合优化 → Rank-1/mAP 评估与预测输出。

### 3.2 数据说明与预处理策略（简要）
根据导师要求，本文不展开底层采集与完整信号处理代码。原始数据由导师团队提供并完成基础处理，本文使用现成点云序列样本进行模型研究。本文侧重说明与实现如下工作：
1. 按身份目录组织样本（如 p_1~p_30）。  
2. 训练/评估时进行在线规格化：固定帧数、固定每帧点数、固定特征维。  
3. 对异常读取样本进行兜底处理，保证训练流程可持续。  

### 3.3 网络结构设计
1. 空间分支：PointNet 提取单帧空间几何特征。  
2. 时序分支：LSTM 建模跨帧动态变化。  
3. 聚合层：Mean Pooling 将序列特征聚合为固定长度身份表示。  
4. 监督头：分类器输出类别 logits，用于 CE 损失计算。  

### 3.4 训练、评估与预测流程设计
系统支持四种模式：
1. train：训练模型并周期评估，自动保存最佳模型。  
2. eval：加载已训练模型，输出 Rank-1、Rank-3、Rank-5 与 mAP。  
3. predict：单样本预测，输出 top1/top5 身份。  
4. predict_batch：随机抽样批量预测，输出每个样本结果与整体准确率。

## 4 系统实现
### 4.1 开发环境与工程结构
开发语言为 Python，深度学习框架为 PyTorch。核心代码文件为 `python/radar_baseline.py`。工程目录包含配置、模型输出、数据目录与运行脚本等，满足中期与后续答辩的复现展示需求。

### 4.2 核心模块实现
1. 数据模块：加载 `.npy` 样本并执行在线标准化。  
2. 模型模块：RadarFeatureExtractor（PointNet + LSTM + Mean Pooling）。  
3. 损失模块：Triplet Loss + CrossEntropy Loss。  
4. 指标模块：基于检索协议计算 Rank-1/mAP。  

### 4.3 运行模式设计
命令行参数支持 `mode + 参数` 的统一接口，减少重复脚本。通过 `seed`、`split_ratio` 和 `checkpoint_path` 控制实验可复现性与结果对比一致性。

## 5 实验与结果分析
### 5.1 实验设置
1. 数据组织：按身份目录管理样本。  
2. 输入规格：20帧×80点×4维。  
3. 训练策略：AdamW、学习率衰减、梯度累积、早停机制。  
4. 评估指标：Rank-1、Rank-3、Rank-5、mAP。  

### 5.2 训练结果分析
训练过程中记录并可视化 Rank-1 与 mAP 曲线。整体趋势表现为：前期快速提升，中后期增幅减缓并逐步收敛。最佳模型在验证指标提升时自动保存，保证后续 eval/predict 使用一致权重。

### 5.3 评估结果分析
独立评估模式下输出检索指标，可用于最终汇报口径。实验表明，当前纯雷达基线在目标数据上可实现有效身份区分，具备较好的基础识别能力。

### 5.4 预测结果与误差分析
批量预测模式输出每个样本的“真实身份/预测身份/是否正确”，并统计整体准确率。误差主要集中在步态差异较小的个体组合，说明模型在细粒度区分方面仍有改进空间。后续可通过参数微调、损失权重调整与难例采样提升稳定性。
